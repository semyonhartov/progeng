# Development Guide

## Назначение

Этот документ описывает устройство проекта и базовый порядок разработки.

## Стек

- `Python`
- `PySide6`
- `qfluentwidgets`

## Точка входа

- `main.py` запускает приложение, UI и фоновые сервисы

## Основные каталоги

- `core/`:
  бизнес-логика, управление инстансами, процессами, бэкапами, удалённым доступом
- `ui/`:
  главное окно, дашборд сервера, карточки, диалоги
- `resources/`:
  вспомогательные ресурсы приложения

## Ключевые модули

- `core/server_manager.py`:
  модели `ServerInstance`, `BackupConfig`, `BackupSchedule`, хранение конфигурации инстансов
- `core/process_manager.py`:
  запуск, остановка, логи, состояние процесса, usage
- `core/remote_access.py`:
  встроенный HTTP API для внешнего доступа, авторизация, права, удалённые действия
- `core/backup_manager.py`:
  создание, список, восстановление, удаление бэкапов
- `core/mod_manager.py`:
  установка и управление модами
- `ui/main_window.py`:
  список инстансов и подключение внешних серверов
- `ui/server_dashboard.py`:
  вкладки управления инстансом

## Как устроен внешний инстанс

1. На стороне владельца локальный инстанс публикуется через `RemoteAccessService`.
2. Внешний клиент проходит `authenticate()`.
3. После входа клиент хранит локальную запись внешнего инстанса с токеном и правами.
4. Все операции выполняются через `/api/state` и `/api/action`.
5. Внешний инстанс не должен выполнять локальные файловые или процессные действия.

## Правила при доработках внешнего доступа

- не использовать локальный `server.path` как реальный путь к данным удалённого сервера
- не запускать локальные процессы для `is_remote=True`
- не показывать пользователю кнопки, открывающие локальные папки владельца
- проверять права `remote_permissions` перед UI-действиями и на серверной стороне API
- любые ошибки удалённого API возвращать как JSON, а не разрывать HTTP-соединение

## Хранение данных

- конфиг каждого инстанса: `instances/<slug>/administale.json`
- база удалённых пользователей: `remote_access.sqlite3`
- локальные бэкапы: `backups/<slug>/`

## Локальный запуск для разработки

```bash
pip install -r requirements.txt
python main.py
```

## Быстрая проверка после изменений

### Синтаксис

```bash
python -m py_compile main.py core/*.py ui/*.py
```

### Ручные сценарии

1. Создать локальный инстанс.
2. Открыть дашборд и проверить запуск, остановку, консоль и настройки.
3. Создать удалённого пользователя для локального инстанса.
4. Подключить внешний инстанс через главное окно.
5. Проверить статус, логи, команды, моды, файлы и бэкапы через удалённый доступ.

## Что важно не ломать

- сериализацию `ServerInstance.to_dict()` и `from_dict()`
- совместимость локальных инстансов с уже сохранёнными конфигами
- разграничение локального и удалённого поведения в `ui/server_dashboard.py`
- неблокирующее взаимодействие с `AsyncProcessManager`

## Идеи для будущих улучшений

- выделить отдельный клиент для удалённого API
- покрыть `remote_access.py` и `server_dashboard.py` автотестами
- добавить загрузку и скачивание файлов потоками вместо base64 для крупных файлов
- добавить health-check и более явный индикатор потери соединения
